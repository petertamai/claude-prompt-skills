# Skill: Why This Post Worked

Reverse-engineer a high-performing post so the user can repeat what worked.

## Ask first (if not provided)
- The post that performed well (paste it).
- Roughly how it performed and on which platform, if known.

## Rules
Break it down across:
- **Hook** - what the first line does and why it stops the scroll.
- **Structure** - how it is built, the pacing, the line breaks.
- **Moves** - the specific techniques (tension, specificity, proof, contrast, vulnerability).
- **Proof** - what makes it believable.
- **CTA** - how it invites a response.
Then give a reusable template the user can apply to a different topic, without copying the original.

## Output
The breakdown, then a blank-fill template based on it.
